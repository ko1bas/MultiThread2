Реализовать многопоточную обработку файлов предыдущего задания (т.е. в
приложение можно передать несколько файлов для обработки). Все удаленные
слова сохраняются в выходной файл, при этом в файле должны содержаться только
уникальные слова.

Способ запуска:
  java -jar multiThread-0.0.1.jar firstmodule.jar output.txt 1.txt 2.txt 3.txt 4.txt 5.txt
  
Использовал Maven 3.2.3, jdk 8 Update 25, Eclipse Luna.

В общем случае не ясно сколько будет файлов, какого они размера, где находятся, 
влезут ли они в оперативную память и т.д. Поэтому, после проверки аргументов командной строки, я анализирую 
количество доступных ресурсов (оперативная память, кол-во ядер процессора) и предполагаемый фронт работ (количество и размер файлов)
Исходя из этих параметров, хитрой эвристикой инициализирую нужными параметрами структуры, необходимые для решения задачи,
И выбираю реализацию этих структур. 
В частности, если анализ показал, что файлы не помещаются в оперативную память, использую хранилище на основе БД.
Иначе - на основе ConcurentHashMap.

Пул из N потоков читает доступные файлы и записывает в блокирующуюся очередь.  
Пул из M потоков читает строки из очереди, получает удаленные из строки слова и записывает их в общее хранилище.
Когда все потоки завершают свою работу, записываю все уникальные удаленные слова в файл.

Как мне кажется, разделение io-потоков и вычислительных потоков позволит более гибко конфигурировать производительность
программы для разных случаев.

Описание поставки:
  Папка common содержит общие классы и интерфейсы, которые используют другие проекты.
  Папка firstModule реализует загружаемый плагин.
  Папка secondModule реализует загружаемый плагин. 
  Папка MultiThread2 содержит проект, который решает задачу.
  Папка Jars содержит все необходимые для запуска в командной строке файлы.

Способ запуска:
  java -jar multiThread-0.0.1.jar firstmodule.jar output.txt 1.txt 2.txt 3.txt 4.txt 5.txt
  
Использовал:
  Maven 3.2.3, jdk 8 Update 25, Eclipse Luna.
